## Context

AdminPage.tsx는 단일 파일(~350줄)로, Supabase RPC 2~3개를 호출해 사용자/채널 데이터를 가져오고 테이블(PC)/카드(모바일)로 렌더링한다. 로직(fetch, toggle, delete)은 이미 완성되어 있고, 이번 변경은 **순수 UI/레이아웃 개선**이다.

현재 문제:
1. PLAN_LABELS 매핑은 코드에 있으나 배포 반영 여부 불확실 → 점검 필요
2. PC 테이블 패딩이 과도해 한 화면에 3~4명만 보임
3. "마스터 액션" 이라는 컬럼명이 직관적이지 않음
4. 채널 컬럼이 존재하나 사용자에게 잘 보이지 않음
5. 정렬이 일관되지 않음 (채널 컬럼은 text-left, 상태는 text-center 등 혼재)

## Goals / Non-Goals

**Goals:**
- AdminPage.tsx의 JSX/className만 수정하여 레이아웃 고도화
- 플랜 라벨이 실제 배포에서 Free/Basic/Pro/School/Event로 표시되는지 확인
- PC 테이블을 컴팩트하게 만들어 한 화면에 더 많은 사용자 표시
- 액션 버튼에 텍스트 라벨을 추가해 기능을 명확히 전달
- 채널 확장/삭제 UI가 눈에 잘 띄도록 개선

**Non-Goals:**
- DB RPC 함수 변경 없음
- 새로운 기능 추가 없음 (정렬, 필터링 등)
- 비즈니스 로직 변경 없음

## Decisions

### 1. PC 테이블 컴팩트화
- 행 패딩: `py-6 lg:py-8` → `py-4`
- 셀 패딩: `px-6 lg:px-10` → `px-4 lg:px-6`
- 이메일 아이콘: `w-12 h-12` → `w-9 h-9`, 아이콘 크기도 축소
- **이유**: 한 화면에 5~6명 이상 표시 목표

### 2. 컬럼 구조 및 정렬
| 컬럼 | 정렬 | 비고 |
|------|------|------|
| 교사 (Email) | 좌측 | 아이콘 + 이메일 |
| 플랜 | 좌측 | 라벨 + 크레딧 |
| 채널 | 중앙 | 숫자 버튼 (확장 트리거) |
| 가입일 | 좌측 | 날짜 텍스트 |
| 상태 | 중앙 | Active/Blocked 뱃지 |
| 관리 | 우측 | PW 재설정 + 차단/복구 (텍스트 라벨 병기) |

### 3. "마스터 액션" → "관리"
- 컬럼 헤더명 변경
- PC에서 아이콘 + 짧은 텍스트 라벨 (예: Key 아이콘 + "PW", Block 아이콘 + "차단")
- 모바일에서는 기존처럼 아이콘만 (공간 절약)

### 4. 채널 컬럼 강조
- 채널 수 버튼을 좀 더 눈에 띄는 스타일로 (배경색 + 볼드 숫자)
- 확장 영역에 얇은 상단 보더 + 살짝 들여쓰기

### 5. 모바일 카드 레이아웃
- 기존 구조 유지하되 채널 버튼을 액션 버튼 줄에 함께 배치
- 플랜 뱃지를 좀 더 눈에 띄게 (배경색 추가)

## Risks / Trade-offs

- [컬럼 수 6개] PC 1024px 이하에서 가로 스크롤 가능성 → 채널/가입일 컬럼을 최소 너비로 설정하여 대응
- [텍스트 라벨 추가] 관리 컬럼 너비 증가 → 짧은 축약어 사용 (PW, 차단/복구)
