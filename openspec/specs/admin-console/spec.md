## ADDED Requirements

### Requirement: 플랜 라벨은 구독 페이지와 동일하게 표시
어드민 페이지에서 사용자 플랜을 표시할 때, DB 값을 다음 매핑으로 변환하여 표시 SHALL:
- `free` → Free, `pro` → Basic, `plus` → Pro, `school` → School, `tester`/`test`/`event` → Event

#### Scenario: pro 플랜 사용자가 어드민에 표시됨
- **WHEN** DB에서 plan='pro'인 사용자가 조회됨
- **THEN** 어드민 페이지에서 "Basic"으로 표시됨

#### Scenario: 매핑에 없는 플랜값
- **WHEN** DB에서 알 수 없는 plan 값이 조회됨
- **THEN** 원본 값을 그대로 표시함

### Requirement: PC 테이블 컬럼 정렬 규칙
PC 테이블의 각 컬럼은 다음 정렬을 따름 SHALL:
- 교사(Email), 플랜/크레딧, 가입일자: 좌측정렬
- 채널: 중앙정렬
- 계정 상태: 중앙정렬
- 관리: 우측정렬

#### Scenario: PC에서 테이블 렌더링
- **WHEN** 어드민 페이지가 PC(md 이상)에서 렌더링됨
- **THEN** 이메일·플랜·가입일은 좌측정렬, 채널·상태는 중앙정렬, 관리 버튼은 우측정렬로 표시됨

### Requirement: 관리 컬럼 직관성 개선
"마스터 액션" 헤더를 "관리"로 변경하고, PC에서 아이콘 옆에 짧은 텍스트 라벨을 표시 SHALL:
- Key 아이콘 + "PW" 텍스트
- Block/Unblock 아이콘 + "차단"/"복구" 텍스트

#### Scenario: PC에서 관리 버튼 표시
- **WHEN** PC 테이블에서 active 상태 사용자의 관리 컬럼을 봄
- **THEN** Key 아이콘 옆에 "PW" 텍스트, UserX 아이콘 옆에 "차단" 텍스트가 표시됨

#### Scenario: 모바일에서 관리 버튼 표시
- **WHEN** 모바일 카드에서 관리 버튼을 봄
- **THEN** 아이콘만 표시됨 (텍스트 없음, 공간 절약)

### Requirement: 채널 컬럼 표시 및 확장
각 사용자 행에 채널 수 버튼이 표시되며, 클릭 시 해당 사용자의 채널 목록이 확장됨 SHALL. 각 채널에는 이름, 학생 수, 삭제 버튼이 표시됨.

#### Scenario: 채널 수 버튼 클릭
- **WHEN** 사용자 행의 채널 수 버튼을 클릭
- **THEN** 해당 행 아래에 채널 목록이 확장되어 표시됨

#### Scenario: 채널 삭제
- **WHEN** 확장된 채널 목록에서 삭제 버튼 클릭 후 확인
- **THEN** 해당 채널이 삭제되고 목록에서 제거됨

### Requirement: PC 테이블 컴팩트 레이아웃
PC 테이블의 행 높이와 셀 패딩을 줄여 한 화면에 더 많은 사용자를 표시 SHALL.

#### Scenario: PC 테이블 데이터 밀도
- **WHEN** 어드민 페이지가 PC에서 렌더링됨
- **THEN** 셀 패딩이 `py-4 px-4 lg:px-6`으로 설정되어 기존보다 컴팩트하게 표시됨

### Requirement: 모바일 카드 레이아웃 개선
모바일 카드에서 채널 버튼이 액션 버튼과 같은 줄에 배치되어 눈에 잘 띄도록 표시 SHALL.

#### Scenario: 모바일 카드 렌더링
- **WHEN** 어드민 페이지가 모바일에서 렌더링됨
- **THEN** 각 사용자 카드에 이메일, 플랜, 상태, 가입일, 채널 수, 관리 버튼이 표시되고, 채널 수 클릭 시 확장됨
